# 健康人群分组评估

## 概述

根据客户健康档案数据（基础信息、体检指标、病史等），综合评估后将用户归入以下四类：

| 分组 | 说明 |
|------|------|
| **健康** | 所有指标在正常范围，无已知疾病 |
| **亚健康** | 存在1-2项指标轻度异常，无确诊慢性病 |
| **慢病** | 已确诊慢性病（高血压、糖尿病、高血脂等），或多项指标显著异常 |
| **重症** | 严重疾病状态，多项指标严重异常，或存在严重并发症 |

## 必采指标

以下指标为分组评估的必要输入（与其他 skill 共用 health-basic 问卷）：

| 字段 | 说明 | 来源 |
|------|------|------|
| `gender` | 性别 | 问卷 |
| `age` | 年龄 | 问卷 |
| `height` | 身高 (cm) | 问卷 |
| `weight` | 体重 (kg) | 问卷 |

## 推荐指标

越多指标越准确，以下为加分项：

| 字段 | 说明 | 参考范围 |
|------|------|----------|
| `systolic_bp` / `diastolic_bp` | 血压 (mmHg) | <140 / <90 |
| `fasting_glucose` | 空腹血糖 (mmol/L) | 3.9-6.1 |
| `hba1c` | 糖化血红蛋白 (%) | <6.5 |
| `total_cholesterol` | 总胆固醇 (mmol/L) | <5.2 |
| `ldl_c` | 低密度脂蛋白 (mmol/L) | <3.4 |
| `hdl_c` | 高密度脂蛋白 (mmol/L) | ≥1.0 |
| `tg` | 甘油三酯 (mmol/L) | <1.7 |
| `uric_acid` | 尿酸 (μmol/L) | 男<420, 女<360 |
| `bmi` | BMI (kg/m²) | 18.5-23.9 |
| `creatinine` | 肌酐 (μmol/L) | 男44-133, 女44-97 |
| `alt` | 谷丙转氨酶 (U/L) | <40 |
| `disease_labels` | 已确诊疾病标签列表 | — |
| `disease_history` | 病史信息 | — |

## 分组规则

分类逻辑由脚本实现，核心步骤：

1. **收集所有可用指标** — 从健康档案、问卷、请求参数中提取
2. **按指标异常程度打分** — 每项指标有异常阈值和权重
   - 轻度异常（如BMI 24-27.9）：低分
   - 中度异常（如收缩压 140-159）：中分
   - 重度异常（如收缩压 ≥180 或空腹血糖 ≥11.1）：高分
3. **结合病史信息** — `disease_labels` 中包含慢性病诊断直接加分
4. **综合评分映射到四个分组**：
   - 0-20 分 → **健康**
   - 21-50 分 → **亚健康**
   - 51-80 分 → **慢病**
   - 81+ 分 → **重症**

## 改善建议生成规则

根据人群分组结果生成个性化改善建议，输出 JSON 数组，每项格式：
```json
{"type": "diet|exercise|sleep|monitoring|medication", "title": "处方标题", "content": ["建议1", "建议2"], "priority": "high|medium|low"}
```

### 生成规则

1. **健康人群** → 以维持健康为主
   - 饮食：均衡饮食，控制钠盐<6g/天，增加蔬果
   - 运动：每周150分钟中等强度有氧运动
   - 睡眠：保证7-8小时规律睡眠
   - 监测：每年体检

2. **亚健康人群** → 针对异常指标干预
   - 饮食：针对性调整（如BMI偏高则控制热量）
   - 运动：每周150-300分钟有氧运动
   - 睡眠：规律作息
   - 监测：每6个月复查异常指标

3. **慢病人群** → 规范化管理
   - 饮食：根据具体疾病调整（如高血压限盐、糖尿病控糖）
   - 运动：中等强度运动，避免剧烈运动
   - 监测：每1-3个月复查，遵医嘱用药
   - 药物：遵医嘱规范用药

4. **重症人群** → 综合干预+专科就诊
   - 饮食：严格饮食管理
   - 运动：低强度运动为主，医嘱指导下进行
   - 监测：每1-3个月复查+专科随访
   - 药物：多学科联合管理，请遵医嘱

## 问卷复用

本 skill 复用 health-basic 问卷（gender、age、height、weight），与其他风险评估 skill 共用同一套基础信息采集流程。
